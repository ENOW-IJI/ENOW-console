Introduction

ENOW-console is a console(with dashboard) for huge Internet of Things environment platform. As the future scheme it will support Raspberry Pi for controlled IoT device via MQTT and webhooks for connecting other services and server via REST APIs over HTTP. When it comes to IoT, hard to find integrated development environment platform and yes, we need this kind of platform eventually. However, if we could make our own development environment for IoT, could it surely deal with complex system and bunch of data properly? you probably consider using distributed realtime computation system like Apache Kafka and Apache Storm for huge IoT system.

Stages

The stage page consists of four major functions:

  • RoadMap Roadmap Editor. You can create your own recipe by adding devices, codes.
  • Load Recipes You can load current running Roadmaps or all lists of Roadmap recipes you made by connecting to MongoDB.
  • Add Device Add one device to current Roadmap. Added window is called 'node'.
  • Save Recipe Save current Roadmap to MongoDB://enow/recipes.



In Image, there are three nodes. In nodes, they have two circles. Empty circle means 'start point' and filled circle means 'end point'.
Each nodes can be connected to others by dragging 'start point' to 'end point'. If connection is created, data will flow as arrow direction.
Connections can be like 1:1 or 1:n even n:n is possible.

Attention!

Independent node can't have connections like n:1 or n:n. Only 1:1 or 1:n can be accepted.






Nodes has some options like below.

Device Options

  • Device/Lambda Select device or lambda. Device is subjected to broker. Lambda is just a function to get values or send values to ohter nodes.
  • Independent If you want to trigger this Roadmap with active sensor value, turn off this switch. Otherwise, turn on the switch.
  • Broker List Choose Broker that you added in 'settings' menu.
  • Save Save Device settings.
  • Edit code Open Code Editor.

Analytics

  • Traffic Chart Show traffics of whole system.
  • Log Chart Show the number of logs that is send from Apache Kafka to Server.
  • ErrorRate Chart Show error rate.
    'Passed' means 'no error'.
    'ConnectionError' means 'connection failed'. Check MongoDB or Apache Kafka connection.
    'TypeError' means 'code is not a json type.'. Check code editor to find the problem.

AdminLTE supports the following browsers:

  • IE 9+
  • Firefox (latest)
  • Safari (latest)
  • Chrome (latest)

Note: IE9 does not support transitions or animations. The template will function properly but it won't use animations/transitions on IE9.

FAQ

Can ENOW-console be used with Wordpress?

ENOW-console is an HTML template that can be used for any purpose. However, it is not made to be easily installed on Wordpress. It will require some effort and enough knowledge of the Wordpress script to do so.

Is there an integration guide for PHP frameworks such as Yii or Symfony?

Short answer, no. However, there are forks and tutorials around the web that provide info on how to integrate with many different frameworks. There are even versions of AdminLTE that are integrated with jQuery ajax, AngularJS and/or MVC5 ASP .NET.

How do I get notified of new ENOW-console versions?

The best option is to subscribe to our mailing list using the subscription form on Almsaeed Studio. If that's not appealing to you, you may watch the repository on Github or visit Almsaeed Studio every now and then for updates and announcements.
